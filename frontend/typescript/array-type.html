数组类型 ​

在 TypeScript 中，数组类型有多种定义方式，比较灵活，也是最常用的类型之一。

「类型 + 方括号」表示法 ​

定义一个数组，可以定义字符串，数值，布尔值，任意类型的数组。

ts
let strAry: string[] = ['A', 'B', 'C']
let numArr: number[] = [1, 2, 3]
let booAry: boolean[] = [true, false]
let anyAry: any[] = ['A', 1, true]
let unknownAry: unknown[] = ['A', 1, true]

泛型数组 ​

泛型定义，可以定义字符串，数值，布尔值，任意类型的数组。

ts
let strAry: Array<string> = ['A', 'B', 'C']
let numAry: Array<number> = [1, 2, 3]
let booAry: Array<boolean> = [true, false]
let anyAry: Array<any> = ['A', 1, true]
let unknownAry: Array<unknown> =['A', 1, true]

元组 Tuple ​

数组是日常开发大量使用的数据结构，但在某些情况下，使用元组 (Tuple) 来代替数组要更加妥当，比如一个数组中只存放固定长度的变量，但进行了超出长度地访问：

ts
const arr: string[] = ['A', 'B', 'C']
console.log(arr1[9])
1
2

这种情况肯定是不符合预期的，因为能确定这个数组中只有三个值，并希望在越界访问时给出类型报错，但这里没有，这时可以使用元组类型进行类型标注：

ts
const arr: [string, string, string] = ['A', 'B', 'C']
console.log(arr[9])
// 报错：长度为 "3" 的元组类型 "[string, string, string]" 在索引 "9" 处没有元素。
1
2
3

除了同类型的元素以外，元组内部也可以声明多个与其位置强绑定的，不同类型的元素：

ts
const arr: [string, number, boolean] = ['Zzai', 9, true]
1

在这种情况下，对数组合法边界内的索引访问（即 0、1、2）将精确地获得对应位置上的类型。

同时元组也支持了在某一个位置上的可选成员：

ts
const arr: [string, number?, boolean?] = ['Zzai']

// 下面这么写也可以
// const arr: [string, number?, boolean?] = ['Zzai', , ,]
1
2
3
4

对于标记为可选的成员，在 strictNullChecks 配置下会被视为一个 string | undefined 的类型。此时元组的长度属性也会发生变化，比如上面的元组，其长度的类型可能为 1 | 2 | 3

ts
type TupleArrLength = typeof arr.length // 1 | 2 | 3
1

暂时了解：类型别名 (type)、类型查询 (typeof) 以及联合类型。

具名元组 ​

考虑到某些拼装对象太麻烦，完全可以使用具名元组来做简单替换。为元组中的元素打上类似属性的标记。

ts
const arr: [name: string, age: number, male: boolean] = ['Zzai', 18, true]
1

具名元组可选元素的修饰符将成为以下形式：

ts
const arr: [name: string, age: number, male?: boolean] = ['Zzai', 599, true]
1

实际上除了显式地越界访问，还可能存在隐式地越界访问，如通过解构赋值的形式：

ts
const arr: string[] = []
const [ele1, ele2, ...rest] = arr
1
2

此时仍然无法检查出是否存在隐式访问，因为类型层面并不知道它到底有多少个元素。但对于元组，隐式的越界访问也能够被揪出来给一个警告。

ts
const fruits: [string, number, boolean] = ['苹果', 9, true]

// 长度为 "3" 的元组类型 "[string, number, boolean]" 在索引 "3" 处没有元素。
const [apple, quantity, tasty, other] = fruits
1
2
3
4

总之，使用元组能够我开发者进一步提升数组结构的严谨性，包括基于位置的类型标注、避免出现越界访问等等。

函数参数为基本类型数组 ​

ts
function ary1(...arg: string[]): string[] {
  return arg
}
console.log(ary1('A', 'B', 'C'))


function ary2(...arg: number[]): number[] {
  return arg
}
console.log(ary2(1, 2, 3))


function ary3(...arg: boolean[]): boolean[] {

arguments 类数组 ​

类数组 (Array-like Object) 具有类似数组结构的对象，它们具有一些类似于数组的特性，但不是真正的数组。

ts
function sum() {
    // 类型“IArguments”缺少类型“number[]”的以下属性: pop, push, concat, join ...
    let args: number[] = arguments
}
1
2
3
4

不应该用普通的数组的方式来描述，而应该用接口。

ts
function sum() {
  let args: {
    [index: number]: number
    length: number
    callee: Function
  } = arguments
}
1
2
3
4
5
6
7

除了约束当索引的类型是数字时，值的类型必须是数字之外，也约束了它还有 length 和 callee 两个属性。

事实上常用的类数组都有自己的接口定义，如 IArguments，NodeList，HTMLCollection 等。

ts
function sum() {
    let args: IArguments = arguments
}
1
2
3

其中 IArguments 是 TypeScript 中定义好了的类型，它实际上就是：

ts
interface IArguments {
    [index: number]: any
    length: number
    callee: Function
}
